Rust meets the web – a clash of programming paradigms (2020)

shwestrick 2021-06-14 14:46:53 +0000 UTC [ - ]

In the "Race Conditions? Not with Javascript!" section:

> They [...] decided that no concurrency between threads is allowed, for the safety of everybody. This resolves the race conditions by removing the possibility that multiple threads ever run at the same time.

Nope! It's very easy to write a JS program with a race condition. Like this one, which sometimes outputs 0, sometimes 1:

    var a = 0;
    setTimeout(function() { a = 1; }, 50);
    setTimeout(function() { console.log(a); }, 49);
